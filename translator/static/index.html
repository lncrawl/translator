Language detection

    
Local detection — costs no engine quota, one line per text
      (POST /detect).

    
    

      
      
    

    

  







  

    
Engines

    
Live status of every configured engine: routing eligibility,
      rate-limit cooldowns, and quota state (GET /engines). The
      enable switch is a config write and needs the admin token.

    

      
      
    

    

  







  

    
Configuration

    
The live runtime config (GET /config). Writes
      apply atomically, persist to config.yml, and need the admin
      token. Secrets never appear here — providers reference API keys by env
      var name only.

    

      
      
    

  


  

    
Providers

    
An API account: base URL, credential env var, and account-wide
      rate limits shared by all of its engines.

    

    

  


  

    
Engines

    
One model on one provider — what the routing lanes reference.

    

    

  


  

    
Routing lanes

    
Priority order per request type; the router walks the lane and
      uses the first engine that is enabled, keyed, and not cooling down.
      Engines without a working key are not offered here.

    

      

      

    

    

      
      
    

  


  

    
Failure policy

    
Retry, fallback, and cooldown behavior of the router.

    

      
      
      
      
    

    

  


  

    
Raw config

    
The full config as JSON. Apply replaces the whole config after
      server-side validation (PUT /config).

    
    

      
    

  








  



  
Access tokens

  
Stored only in this browser (localStorage). The API token is
    needed when the server sets AUTH_TOKEN; config writes need
    ADMIN_TOKEN (the API token is used as fallback).

  
  
  

    
    
  





  
Provider

  
  

    
    
  

  
  
The key is stored in the config file; engines on
    this provider stay disabled until it is set.
